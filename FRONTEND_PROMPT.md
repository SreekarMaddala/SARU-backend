# Frontend Implementation Prompt: Feedback CSV Upload

## Overview
Implement a feedback CSV upload feature in the frontend that works with the updated backend API. The backend no longer uses `product_id` — products are now identified by `(company_id, model_number)` composite key.

---

## API Endpoint

```
POST /feedback/upload_csv
Content-Type: multipart/form-data
Authorization: Bearer <JWT_TOKEN>
```

### Request
- **file**: CSV file (multipart/form-data)

### Response (200 OK)
```json
{
  "inserted": 42,
  "rejected_missing_email_and_mobile": 3,
  "message": "Sentiment, topics, and sentiment_score were auto-generated by the NLP model."
}
```

### Error Responses
- **400 Bad Request**: Missing required columns, empty file, invalid product_model_number, etc.
- **401 Unauthorized**: Invalid or missing JWT token

---

## CSV Format Requirements

### Required Columns
| Column | Description | Example |
|--------|-------------|---------|
| `channel` | Source channel | `email`, `twitter`, `google_forms`, `support_chat` |
| `text` | Feedback content | `Great product, loved the features!` |

### Required (at least one per row)
| Column | Description | Example |
|--------|-------------|---------|
| `email` | Customer email | `john@example.com` |
| `mobile` | Customer phone | `+1234567890` |

### Optional Columns
| Column | Description | Example |
|--------|-------------|---------|
| `name` | Customer name | `John Doe` |
| `product_model_number` | Product model number | `IPHONE-15-PRO` |

### ⚠️ Important Notes
- **DO NOT** include `product_id` — it has been replaced with `product_model_number`
- **DO NOT** include `sentiment`, `sentiment_score`, or `topics` — these are auto-generated by backend NLP
- **DO NOT** include `id` — auto-generated by database
- Each row must have at least `email` OR `mobile` (or both)
- `product_model_number` must match an existing product in the current company's catalog

### 2. Features to Implement

#### A. File Selection
- Accept only `.csv` files
- Show selected filename
- Validate file size (e.g., max 10MB)

#### B. Template Download
- Provide a downloadable CSV template with correct headers:
  ```
  channel,text,name,email,mobile,product_model_number
  ```

#### C. Pre-Upload Validation (Optional but recommended)
- Parse CSV client-side to check:
  - Has `channel` and `text` columns
  - Has at least one of `email` or `mobile`
  - Show preview of first 5 rows

#### D. Upload Progress
- Show loading spinner during upload
- Disable upload button while processing

#### E. Success State
- Show `inserted` count
- Show `rejected_missing_email_and_mobile` count
- Display the backend message about auto-generated fields

#### F. Error State
- Display backend error message clearly
- Common errors to handle:
  - "CSV must contain required columns: channel, text"
  - "CSV must contain at least one of: email, mobile"
  - "Invalid product_model_number: XXXXX"
  - "CSV file is empty"

## State Management Example (React-style pseudocode)

```javascript
const [file, setFile] = useState(null);
const [uploading, setUploading] = useState(false);
const [result, setResult] = useState(null);
const [error, setError] = useState(null);

const handleUpload = async () => {
  if (!file) return;
  
  setUploading(true);
  setError(null);
  setResult(null);
  
  const formData = new FormData();
  formData.append('file', file);
  
  try {
    const response = await fetch('/feedback/upload_csv', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${token}`
      },
      body: formData
    });
    
    const data = await response.json();
    
    if (!response.ok) {
      throw new Error(data.detail || 'Upload failed');
    }
    
    setResult({
      inserted: data.inserted,
      rejected: data.rejected_missing_email_and_mobile,
      message: data.message
    });
  } catch (err) {
    setError(err.message);
  } finally {
    setUploading(false);
  }
};
```

## Migration Notes for Existing Frontend

If your frontend previously used `product_id`:

### ❌ Old (REMOVED)
```csv
channel,text,product_id
email,Great product!,123
```

### ✅ New
```csv
channel,text,product_model_number
email,Great product!,IPHONE-15-PRO
```

### Changes Needed:
1. Update CSV template headers: replace `product_id` with `product_model_number`
2. Update any product dropdown from `value={product.id}` to `value={product.model_number}`
3. Update product API calls from `/products/{product_id}` to `/products/{model_number}`
4. Update product list display to show `model_number` instead of `id`
